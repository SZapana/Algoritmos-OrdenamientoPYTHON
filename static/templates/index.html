Algoritmos de Ordenamiento

Bubble Sort

Mejor

O(n)

Promedio

O(n²)

Peor

O(n²)

Compara elementos adyacentes y los intercambia si están en el orden incorrecto.

  • Simple de implementar
  • Bueno para pequeños conjuntos
  • Ineficiente para grandes conjuntos

Insertion Sort

Mejor

O(n)

Promedio

O(n²)

Peor

O(n²)

Construye la secuencia final un elemento a la vez insertando cada elemento en su posición correcta.

  • Eficiente para pequeños conjuntos
  • Estable (no cambia orden iguales)
  • Ineficiente para grandes conjuntos

Selection Sort

Mejor

O(n²)

Promedio

O(n²)

Peor

O(n²)

Divide el arreglo en una parte ordenada y otra no ordenada, seleccionando repetidamente el mínimo elemento.

  • Simple implementación
  • Bajo uso de memoria
  • Lento para grandes conjuntos

Merge Sort

Mejor

O(n log n)

Promedio

O(n log n)

Peor

O(n log n)

Divide el arreglo en mitades, las ordena recursivamente y luego combina las mitades ordenadas.

  • Eficiente para grandes conjuntos
  • Estable (no cambia orden iguales)
  • Requiere espacio adicional

Quick Sort

Mejor

O(n log n)

Promedio

O(n log n)

Peor

O(n²)

Selecciona un 'pivote', particiona el arreglo alrededor del pivote, y ordena recursivamente las particiones.

  • Muy eficiente en la práctica
  • Ordenamiento en el lugar
  • Peor caso O(n²) (raro)
